# MNIST 数据集的图片有非常严格的制作标准，而直接拍摄或手绘的图片往往不符合这些标准：
# 1.笔迹粗细不一：MNIST 的数字笔触较粗且均匀。如果你用圆珠笔写字，线条太细，缩放到 28x28 后可能就断断续续或看不清了。
# 2.没有居中：MNIST 的数字是重心居中的，且四周留有固定的边距。如果你手写的数字偏左、偏右，或者占满了整个画布，模型会完全“看不懂”。
# 3.噪声干扰：拍照会有阴影、纸张纹理，简单的二值化（反色）可能无法完美去除背景，导致输入全是噪点。
# 4.比例失真：直接将长方形图片强制 resize 到 28x28 会导致数字被压扁或拉长。
